# Service Banner Block Builder

Генерирует готовый CS-Cart HTML-блок (баннер + модальная форма) для страниц ремонта Maxmobiles.

## Обязательные входные данные

Перед генерацией убедись, что пользователь предоставил **все 4 пункта**. Если чего-то нет — попроси:

| # | Что | Пример |
|---|-----|--------|
| 1 | **Заголовок** (H1) | `Ремонт iPhone 15 Pro Max` |
| 2 | **Текст под заголовком** (1–3 предложения) | `Любая поломка телефона...` |
| 3 | **URL картинки** | `https://maxmobiles.ru/images/.../iphone-15.webp` |
| 4 | **Услуги** — для каждой: название, URL страницы товара, время ремонта, гарантия, опция (если несколько вариантов на одной странице) | см. формат ниже |
| 5 | **CAT_ID** — ID категории CS-Cart для данного устройства (смотри в URL категории в админке) | `2260` |

### Формат услуг

```
Название: Замена стекла
URL: /displey-zamena-razbitogo-stekla-iphone-15-pro-max/
Время: 30 мин
Гарантия: 360 дней
Опция: (пусто — если на странице одна цена)
Цена: (пусто — берётся из API; «уточняйте» — если цена не показывается)
От: (пусто — фиксированная цена; «да» — если цена «от N ₽»)
```

```
Название: Замена дисплея (оригинал)
URL: /displey-original-zamena-iphone-15-pro-max/
Время: 40 мин
Гарантия: 360 дней
Опция: Оригинал   ← должна точно совпадать с variant_name в CS-Cart API
```

```
Название: Чистка после попадания воды
URL: /profilaktika-posle-vody-cena-ot-iphone-15-pro-max/
Время: 60 мин
Гарантия: 30 дней
Опция: (пусто)
От: да   ← перед ценой в баннере и в модалке отображается оранжевый «от»
```

```
Название: Замена слухового динамика
URL: /dinamik-vyzova-zvonok-gromkaya-svyaz-zamena-iphone-15-pro-max/
Время: 30 мин
Гарантия: 360 дней
Опция: (пусто)
Цена: уточняйте   ← цена из API НЕ парсится, в баннере отображается «Уточняйте»
```

`Опция` — пустая строка если URL страницы содержит одну цену. Иначе — точное название варианта в CS-Cart (`Копия`, `Оригинал`, `Оригинал "б/у"` и т.д.).

`Цена: уточняйте` — спецрежим. Используй когда конкретная цена не выставляется в каталоге CS-Cart, а сообщается клиенту индивидуально (после диагностики, по запросу и т.п.). При наличии этого флага:
- В объекте SERVICES добавляется `askPrice: true`
- `price` и `priceNum` ставятся равными `'Уточняйте'`
- JS-гидратация **пропускает** эту услугу (не делает API-запрос её цены, не перезаписывает значение)
- В баннере показывается надпись `Уточняйте` без символа `₽` (currency-span автоматически скрывается через `display: none`)
- В модальной форме в поле «стоимость» уходит строка `Уточняйте` — менеджер видит, что нужно согласовать цену

## Процесс генерации

Прочитай файл-эталон:
`c:\Users\Алекс\Service MM\Новые категории ремонта\iphone-15-pro-max-block-1.html`

Затем выполни все замены по таблице из [template-reference.md](template-reference.md).

### Шаги

1. **Определи префикс** — короткий уникальный идентификатор CSS/JS для блока.  
   Правило: `mm` + транслит первых букв устройства, макс. 6 символов.  
   Примеры: `mmb1` (iPhone 15 Pro Max), `mmi14` (iPhone 14), `mmmac` (MacBook).  
   Замени `mmb1` везде в файле на выбранный префикс.

2. **Подставь данные** по таблице замен (см. template-reference.md).

3. **Сформируй массив SERVICES** в JS:

```javascript
var SERVICES = [
  {
    name:     'НАЗВАНИЕ_УСЛУГИ',
    price:    '— ₽',
    priceNum: '—',
    time:     'ВРЕМЯ',
    warranty: 'Гарантия N дней',
    url:      '/URL-СТРАНИЦЫ-ТОВАРА/',
    option:   'ОПЦИЯ'
  },
  {
    name:      'УСЛУГА_С_ЦЕНОЙ_ОТ',
    price:     '— ₽',
    priceNum:  '—',
    fromPrice: true,
    time:      'ВРЕМЯ',
    warranty:  'Гарантия N дней',
    url:       '/URL-СТРАНИЦЫ-ТОВАРА/',
    option:    ''
  },
  {
    name:     'УСЛУГА_БЕЗ_ФИКСИРОВАННОЙ_ЦЕНЫ',
    price:    'Уточняйте',
    priceNum: 'Уточняйте',
    time:     'ВРЕМЯ',
    warranty: 'Гарантия N дней',
    url:      '/URL-СТРАНИЦЫ-ТОВАРА/',
    option:   '',
    askPrice: true
  }
];
```

> **Важно:** для обычных услуг `price` и `priceNum` — **всегда** `'— ₽'` и `'—'`. Конкретные цены в SERVICES не хранятся.
> Хардкодные значения вызывают мигание (flash) при несовпадении со значением из API.
> Цены в block-1 не влияют на SEO (не попадают в сниппет), поэтому статические значения не нужны.
>
> **Критично:** HTML-спан `<span id="ПРЕФИКС-price">` в разметке баннера тоже должен содержать `—`, а не число.
> Это второй источник мигания — браузер рендерит его ещё до выполнения любого JS.
>
> **Спецрежим `fromPrice: true`** (когда пользователь указал «От: да»):
> - `price` и `priceNum` — по-прежнему `'— ₽'` и `'—'` в SERVICES (до гидратации)
> - После гидратации: `s.price = 'от\u00a0' + число + '\u00a0₽'` — с префиксом «от» в строке цены для модалки
> - В HTML перед `<span id="ПРЕФИКС-price">` стоит `<span class="ПРЕФИКС-price-from" id="ПРЕФИКС-price-from">от&nbsp;</span>` со стилем `display:none` по умолчанию
> - В `updateUI` элемент `#ПРЕФИКС-price-from` показывается (`display: 'inline'`) если `s.fromPrice === true`, иначе скрывается (`display: 'none'`). Важно: `''` не работает — CSS-класс оставит элемент скрытым
> - CSS: `.ПРЕФИКС-price-from { font-size: 18px; font-weight: 500; color: #ff9500; display: none; }`
> - JS-гидратация обрабатывает услуги с `fromPrice` в обычном режиме (API-цена парсится), только форматирует `s.price` с префиксом «от»
>
> **Спецрежим `askPrice: true`** (когда пользователь указал «Цена: уточняйте»):
> - `price: 'Уточняйте'`, `priceNum: 'Уточняйте'` — без числа и без знака «₽»
> - `option: ''` — игнорируется, всё равно ничего не парсим
> - JS-гидратация (`mmb1ApplyPriceMap`) **пропускает** услуги с `askPrice: true`
> - В `updateUI` дополнительно скрывается элемент `.ПРЕФИКС-price-currency` (значок ₽), если `s.askPrice === true`, и показывается обратно для обычных услуг; элемент `#ПРЕФИКС-price-from` тоже скрывается
> - Если такая услуга выбрана при загрузке (`_idx === idx`) — в `<span id="ПРЕФИКС-price">` сразу пишем `Уточняйте` в HTML

4. **Пропиши CAT_ID** — ID категории CS-Cart данного устройства:

```javascript
var CAT_ID = XXXX; // ← спросить у пользователя или найти в URL категории в админке CS-Cart
```

5. **Заполни `<select>`** — один `<option>` на каждую услугу.

6. **Форма CS-Cart** — уточни у пользователя `page_id` и ID полей формы (`form_values[N]`), либо используй следующие по порядку после последнего известного. Не выдумывай ID.

7. **Имя файла** — `SLUG-block-1.html`, где SLUG = транслит названия устройства через дефис.  
   Пример: `iphone-14-pro-block-1.html`

8. Сохрани файл в `Новые категории ремонта\`.

## Диагностика — обязательное правило

**Диагностика у нас никогда не бесплатная сама по себе.**
Диагностика бесплатна **только при условии, что клиент соглашается на ремонт**.
Если клиент отказался от ремонта — диагностика оплачивается отдельно.

**Как писать в текстах баннера и модалки:**
- ✅ «Диагностика при вас — быстро и точно»
- ✅ «Точная диагностика при вас»
- ✅ «Профессиональная диагностика — бесплатно при ремонте»
- ✅ «Бесплатная диагностика при проведении ремонта»
- ❌ «Бесплатная диагностика» — без уточнения условия **запрещено**
- ❌ «Бесплатная диагностика при вас» — вводит в заблуждение, **запрещено**

**Trust-item в левой колонке баннера** — обязательная формулировка:
```html
Диагностика при вас — быстро и точно
```
или
```html
Бесплатная диагностика при ремонте
```
Никогда не писать просто «Бесплатная диагностика» без условия.

## Неизменяемые части

Следующие блоки копируй из эталона **без изменений**:

- CSS стили модальной формы (капча, чекбокс, поля, кнопка)
- HTML модальной формы (капча, чекбокс, экран успеха)
- JS логика капчи (`generateCaptcha`, обработчик клика)
- JS маска телефона
- JS динамической загрузки цен через API (`mmb1SlugFromUrl`, `mmb1ApplyPriceMap`, `mmb1ReadCache`, `mmb1WriteCache`, `mmb1LoadPrices`) — менять только `CAT_ID`
- Skip-логика для `askPrice: true` (`if (s.askPrice) return;` в `mmb1ApplyPriceMap`) и переключение `.ПРЕФИКС-price-currency` + `.ПРЕФИКС-price-from` в `updateUI` — копировать **дословно**
- Логика `fromPrice: true`: показ `#ПРЕФИКС-price-from`, форматирование `s.price` с «от» — копировать **дословно**
- CS-Cart атрибуты формы: `action="{""|fn_url}"`, `method="post"`, `enctype="multipart/form-data"`, `name="fake" value=""`
