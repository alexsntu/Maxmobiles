# SEO Meta Builder — Maxmobiles

Generates H1, title, and meta description for **maxmobiles.ru** pages.

## Приоритет трафика: Google лидирует, Яндекс даёт объём заказов

Реальные данные аналитики Maxmobiles:
- **Google**: **56%** органического трафика · средний чек на **20% выше**, чем с Яндекса
- **Яндекс**: **43%** органического трафика · количество заказов примерно равно Google
- **Расчётный вес выручки**: Google ≈ **55%**, Яндекс ≈ **45%**

**Принцип:** оптимизируем под Google в первую очередь (больший трафик + крупные покупки), Яндекс — в приоритете наравне (равный объём заказов). Обе системы критически важны — нельзя жертвовать одной ради другой. Главное — максимальное покрытие обоих поисковиков без нарушения технических требований.

**Формула эффективного веса (на основе доли выручки):**
```
Эффективный вес = (Google-значимость × 0.55) + (Яндекс-значимость × 0.45)
```

## Workflow (execute in order)

### Step 1 — Parse the request

Extract from the user's message:
- **Page type**: конкретная модель / серия / топовая категория / аксессуары / сервис-услуга
  - **Исключение — пагинация / фильтры** (`?page=2`, `?features_hash=...`): скилл не генерирует мета-теги для этих URL. Решение — `canonical` на первую страницу категории или `noindex` на уровне CS-Cart настроек.
- **Product name / category** (e.g., iPhone 16 Pro, MacBook Air M4, AirPods Pro 3, ремонт iPhone)
- **Keyword data**: если пользователь предоставил список ключей с частотностью/значимостью
- **Quantity**: single page or batch

**Маршрут выполнения:**

```
keyword data НЕ предоставлена → Step 2 (4 запроса) → Step 2b → Step 2c → Step 3
keyword data предоставлена    → Step 2-mini (3 запроса) → Step 2b → Step 2c → Step 3
```

**Step 2c (Gap Analysis) выполняется всегда.** При пропущенном основном Step 2 — выполнить **Step 2-mini**: 2 WebSearch параллельно (обязательные):
1. `"купить [Категория/Модель] Крым"` — поиск реальных конкурентов
2. `"[Категория/Модель] Севастополь"` — региональные конкуренты

**Опциональный запрос №3** — только если планируется HTML-блок (shop-page-builder):
3. `"как выбрать [Категория/Модель]"` — FAQ-семена для shop-page-builder

После Step 2-mini — также выписать домены из топ-10 по каждому запросу и проверить наличие приоритетных конкурентов Maxmobiles (полный список — в Step 2). Найденных пометить **★** для Step 2c.

**Batch-режим — стратегия SERP-запросов:**
- **2–4 страницы**: запустить WebSearch для каждой страницы параллельно (N×4 одновременно).
- **5+ страниц**: группировать похожие категории. SERP по «купить iPhone Крым» переиспользовать для всех серий iPhone. Gap Analysis — один общий, если страницы из одной категории.

### Step 2b — Анализ ключевых слов (traffic scoring)

**Когда keyword data не предоставлена — оценка значимости по SERP:**

Если численные данные ключей отсутствуют (SERP-маршрут), оценить значимость по частоте появления ключа в title/H1 реальных конкурентов (не агрегаторов) из Step 2:

| Частота в топ-10 реальных конкурентов | Оценочная значимость |
|---|---|
| 3+ упоминаний | ~50 — кандидат в title |
| 1–2 упоминания | ~20 — кандидат в description |
| 0 упоминаний | < 15 — пропустить |

Пометить в комментарии: «значимость оценена по SERP-частоте, не по keyword data».

**Fallback при малом числе реальных конкурентов (1–2 в топ-10):**
Если реальных конкурентов ≤ 2, порог 3+ не применять. Ключ, присутствующий в title хотя бы одного реального конкурента, считать кандидатом в title (~40). Пометить: «мало реальных конкурентов — оценка по присутствию в title».

### Step 2c — Gap Analysis реальных конкурентов

Выполнять **всегда** (на обоих путях). Двухшаговый процесс: сначала из SERP-сниппетов, затем при необходимости WebFetch.

**Шаг 1 — быстрый анализ SERP-сниппетов (без дополнительных запросов):**
Для каждого реального конкурента из топ-10 извлечь прямо из результатов поиска:
- Title (виден в SERP напрямую)
- Наличие meta description: если сниппет содержит текст — есть; если пусто или показан только URL — отсутствует
- Кириллика в title/сниппете (да/нет)

**Шаг 2 — WebFetch: условно, максимум 2 параллельных запроса:**
Запускать только если в топ-10 найден ★-конкурент. Тогда параллельно:
- **1 WebFetch на URL страницы ★-конкурента** из SERP-результата (не главная страница домена)
- **1 WebFetch на URL категории Maxmobiles** — для извлечения актуального ассортимента моделей в description

Если ★-конкурентов нет → 1 WebFetch на следующего реального конкурента из SERP. WebFetch на Maxmobiles — только при наличии ★-конкурента.

**Приоритет выбора конкурента для WebFetch:**
1. ★-конкурент из приоритетного списка (Step 2), найденный в топ-10
2. Следующий реальный конкурент из SERP (не агрегатор)
3. Если ни одного реального — fallback с агрегатором (см. ниже)

**Fallback при сбое WebFetch:**
Если WebFetch вернул менее 200 символов читаемого текстового контента (заголовки, параграфы, перечни USP — без учёта HTML-тегов и скриптов) — пропустить этого конкурента, перейти к следующему из SERP. Если все конкуренты недоступны — использовать SERP-сниппеты (title + description из поиска) как частичный источник; пометить: «данные из сниппета, не с полной страницы».

**Если реальных конкурентов не найдено (все топ-10 — агрегаторы):**
1. Взять страницу наиболее релевантного агрегатора → зафиксировать пробелы (нет локального сигнала, нет FAQ, нет сервисного центра, нет кириллики)
2. Записать: «Maxmobiles может стать первым специализированным Apple-магазином в топ-10 по данной нише»
3. Вывести рекомендации на основе пробелов агрегатора

Из каждой страницы извлечь:

| Параметр | Что фиксировать |
|---|---|
| H1 / H2 заголовки | Ключевые слова, наличие гео, формат |
| Title (из `<title>`) | Паттерн, длина, структура |
| FAQ-секция | Есть / нет; если есть — перечислить вопросы |

### Step 3 — Генерация мета-тегов

**Цель не шаблон, а покрытие.** Максимально охватить высоковесовые ключи в рамках технических ограничений поисковых систем.

#### Правила по типу страницы

| Тип страницы | H1 — формат | Гео в H1 | «Купить» в title | Трейд-ин в USP |
|---|---|---|---|---|
| Топовая категория | `[Тип товара] Apple` или `[Тип товара] [Линейка]` | Нет | Внутри title | Да |
| Серия моделей | `[Тип товара] [Серия]` | Нет | Внутри title | Да |
| Конкретная модель | `[Тип товара] [Полное название модели]` | Нет | Первое слово в title | Да |
| Аксессуары | `[Тип аксессуара] Apple` | Нет | Внутри title | Нет |
| Сервис / услуга | `[Услуга] в [Город]` | **Да — обязательно** | Не использовать | Нет |

**Специфика USP по типу:**
- **Аксессуары**: убрать трейд-ин и кредит, акцент на совместимости, доставке, гарантии
- **Сервис**: гарантия на работы, оригинальные запчасти Apple, срок ремонта, авторизованный сервисный центр Apple

#### H1 — что делает заголовок сильным

H1 — навигационный заголовок. Пользователь уже на сайте, коммерческие сигналы ему не нужны. Весь коммерческий сигнал — в title и description.

**Правила длины и формата:**
- Диапазон: **15–45 симв.** (проверять явно)
- Если название продукта < 15 симв. — добавить тип устройства перед ним: «Смартфон iPhone» (15 симв.)
- Запрещено: «Купить», «официальный», «авторизованный»

**Позитивные правила — что делает H1 сильным:**

1. **Порядок слов**: тип устройства → бренд/серия → модель
   - ✅ «Смартфоны iPhone» — тип первым, бренд вторым
   - ❌ «iPhone смартфоны» — обратный порядок хуже читается

2. **Полное официальное название** — не сокращать, не переводить на кириллицу:
   - ✅ «Смартфон iPhone 16 Pro Max»
   - ❌ «Смартфон Айфон 16 Про Макс» — кириллика в H1 не нужна

3. **Для серий** — тип + серия, не только серия:
   - ✅ «Смартфоны iPhone 16»
   - ❌ «iPhone 16» — без типа устройства теряет навигационный смысл

4. **Для мультибрендовых категорий** — основной бренд первым:
   - ✅ «Наушники Apple, Samsung»
   - ❌ «Samsung и Apple наушники»

5. **Для сервисных страниц** — услуга + устройство + город:
   - ✅ «Ремонт iPhone в Севастополе»
   - ❌ «Ремонт телефонов» — слишком широко, без устройства и города

#### title — варианты по разным стратегическим осям

**Технические ограничения (жёсткие):**
- Google обрезает после ~65 симв. / 580px
- Yandex показывает до ~70 симв. без обрезки
- Нет keyword stuffing, нет caps, нет `!`, нет цен в ₽
- `Maxmobiles` всегда последним после `|`

**Три стратегические оси для вариантов — варианты должны различаться по оси, а не только по кириллике:**

| Ось | Что варьировать | Пример |
|---|---|---|
| **А — кириллика** | с/без кириллического варианта бренда | «iPhone» vs «iPhone (Айфон)» |
| **Б — коммерческий сигнал** | «купить» vs «цена» vs «гарантия» | «купить в Крыму» vs «цена и гарантия» |
| **В — длина/гео** | Google ≤65 симв. vs Яндекс ≤70 симв.; «в Крыму» vs «в Севастополе» | «в Крыму» vs «в Севастополе» |

**Процесс генерации title:**

1. Взять ключ с максимальным эффективным весом → основа title
2. Построить **2–3 варианта** по разным осям (не только ось А)
3. Для каждого посчитать покрытие ключей и длину
4. Учесть правило первого слова (см. ниже)

**Если все варианты > 65 симв. — порядок усечения:**
1. Убрать кириллику в скобках (если есть) — экономит 8–15 симв.
2. Сократить «в Крыму» до «Крым» — экономит 2 симв.
3. Убрать коммерческий сигнал («купить»), оставить гео
4. Крайний случай: сократить гео до отсутствия — только продукт + Maxmobiles

**Правило первого слова в title:**
- Категории и серии: продукт первым (`iPhone 16 — купить в Крыму...`)
- Конкретная модель: «Купить» первым (`Купить iPhone 16 Pro Max...`)
- Сервис: услуга первой (`Ремонт iPhone в Севастополе...`)

**Вывод вариантов — показывать пользователю все варианты, не выбирать самостоятельно:**

```
Вариант А: «[title]» ([N] симв.) — [ось: кириллика / коммерч. сигнал / длина]
→ Покрывает: [что именно]. Сильно для: [Яндекс / Google / оба].

Вариант Б: «[title]» ([N] симв.) — [ось]
→ Покрывает: [что именно]. Сильно для: [Яндекс / Google / оба].

Вариант В: «[title]» ([N] симв.) — [ось] (если есть)
→ Покрывает: [что именно].

→ Рекомендация: Вариант [X] — [одна фраза, обоснование из Step 2b данных].
```

#### meta description

**Технические ограничения:**
- Google Desktop: ~155 симв. видимый сниппет
- Яндекс расширенный сниппет: до 300+ симв.
- Оптимальная длина: **180–270 симв.**

**AEO-паттерн первого предложения (Featured Snippet и AI Overviews):**

Структура: `[Продукт/Категория] в Maxmobiles — [прямой ответ на коммерческий запрос]`

AI-системы цитируют предложения, которые:
- Начинаются с объекта запроса (не с «Мы», не с «Maxmobiles»)
- Содержат прямой ответ в первых 10 словах
- Включают гео-сигнал (Крым / Севастополь)
- Заканчиваются конкретным фактом, а не только призывом

❌ Слабо: «Maxmobiles предлагает iPhone с доставкой по России.»
✅ Сильно: «iPhone в Maxmobiles — гарантия 1 год, доставка СДЭК, самовывоз в Крыму.»

**Структура description — фиксированный порядок:**

*Для товарных страниц (категория / серия / модель / аксессуары):*
```
[AEO-первое предложение: продукт + USP-1, USP-2, USP-3 + CTA]     ← до ~155 симв.
[Модельный ряд / ключевые характеристики продукта]                  ← приоритет в хвосте
[Кириллические хвосты — если значимость 15–34]                     ← после моделей
[E-E-A-T: Experience («с 2011 года») → Authoritativeness]           ← в конце хвоста
```

*Для сервисных страниц (ремонт, Trade-In):*
```
[AEO-первое предложение: услуга + город + главный USP сервиса]      ← до ~155 симв.
[Перечень работ / услуг: что конкретно делается]                   ← приоритет в хвосте
[Условия сервиса: запчасти, срок, запись]                          ← после перечня работ
[E-E-A-T: опыт мастеров, авторизованный сервисный центр Apple]     ← в конце хвоста
```

Порядок фиксирован: AI-системы цитируют начало хвоста чаще конца.

- Ключи, не попавшие в title → интегрировать в description естественно
- Доставку и самовывоз указывать вместе: «доставка СДЭК, самовывоз в Крыму»
- CTA до 155 симв.: «Заказывайте онлайн» / «Оформите заказ» / «Выбирайте в каталоге»
- Gap-USP из Step 2c: то, чего не упомянули конкуренты — выдвинуть вперёд

**E-E-A-T в description — приоритет при нехватке места:**

| Приоритет | Сигнал | Фраза | Условие |
|---|---|---|---|
| 1 — обязательно | Trustworthiness | «гарантия 1 год», «трейд-ин», «доставка СДЭК» | Всегда, первые 155 симв. |
| 2 — обязательно | Expertise | «старейший Apple-эксперт в Крыму» / «авторизованный сервисный центр Apple» | Всегда, первые 155 или сразу после |
| 3 — при длине > 180 | Experience | «с 2011 года» / «15 лет на рынке Apple-техники» | В хвосте, если влезает |
| 4 — при длине > 220 | Authoritativeness | «старейший Apple-магазин в Крыму» | Только если есть место |

При нехватке места жертвовать 3 и 4, но не 1 и 2.

**Approved USPs:**
- «гарантия 1 год»
- «доставка СДЭК» / «доставка по России через СДЭК»
- «самовывоз в Крыму» / «самовывоз в Севастополе, Симферополе, Ялте»
- «трейд-ин»
- «покупка в кредит» / «кредит»
- «авторизованный сервисный центр Apple»
- «старейший Apple-эксперт в Крыму»
- «с 2011 года» / «15 лет на рынке Apple-техники»

> Запрещено везде: «рассрочка», эмодзи, цены в ₽, `!`
> «официальный» и «авторизованный» — запрещены в H1 и title. В description «авторизованный сервисный центр» допустимо как фактический USP.

**Многобрендовые категории:**
- H1: `[Тип товара] Apple, [Бренд2]` — не более 2 брендов
- title: `[Тип товара] [Бренды] — купить в Крыму | Maxmobiles`
- Остальные бренды → description

#### JSON-LD description — согласование с мета-тегами

При использовании совместно с shop-page-builder:

- **`WebPage.description`** — должен содержать **полное первое предложение** meta description (не обрезать на 155 симв. — предложение должно быть завершённым). Именно его используют Яндекс Нейро и ChatGPT Search для генерации ответов.
- **`LocalBusiness.description`** — 2–3 предложения: специализация + гео + главный USP. Не повторять meta description дословно — дополнить другим углом.
- Пример `LocalBusiness.description`: «Maxmobiles — старейший Apple-эксперт в Крыму с 2011 года. Самовывоз в Севастополе, Симферополе, Ялте. Доставка СДЭК по всей России, трейд-ин, авторизованный сервисный центр Apple.»

### Step 6 — Семантическое ядро (запросы для отслеживания)

**Выводить по запросу.** После Step 5 добавить одну строку:
> Нужно семантическое ядро для Яндекс Вебмастера / GSC?

Генерировать только при подтверждении. Если пользователь явно запросил ядро в исходном сообщении — выводить без вопроса.

**Принцип:** только запросы, подтверждённые SERP-анализом или keyword data. Не изобретать вслепую.

**Формат запросов:** нижний регистр, без знаков препинания. Пример: `купить iphone 16 pro в крыму` — не «Купить iPhone 16 Pro в Крыму». Это стандарт Яндекс Вебмастера и Google Search Console.

**Объём:** итого на страницу **15–25 запросов** (Tier 1 + Tier 2 + Tier 3 суммарно). Больше — неуправляемо в SEO-инструментах.

**Нежелательные интенты — НЕ включать в ядро:**

```
❌ Информационные:       «[продукт] обзор», «[продукт] отзывы», «[продукт] характеристики»,
                         «лучшие [продукт] 2025», «[продукт] vs [другой]»
❌ Сервисные (для товарных страниц): «[продукт] ремонт», «[продукт] прошивка»
❌ Конкурентные бренды:  запросы с упоминанием конкурентов
❌ Недостижимые:         «[продукт]» без гео при 4+ агрегаторах в топ-10
```

Разделять на три уровня:

---

**Tier 1 — Цели 🎯** *(0–1 агрегатор в топ-10 — Maxmobiles может конкурировать)*

Региональные запросы с гео, с условием покупки, кириллика с гео. Минимум 5 запросов.

```
купить [продукт] в крыму

[продукт] трейд-ин крым
[продукт] в кредит севастополь
[продукт] доставка сдэк             ← только если SERP показывает 0–1 агрегатора в топ-10
[кириллика продукт] купить крым
[продукт] сервисный центр севастополь
```

---

**Tier 2 — Мониторинг 👁** *(2–3 агрегатора в топ-10)*

Высокочастотные запросы без гео, широкие категорийные. Минимум 3 запроса.

```
купить [продукт]
[продукт] цена
[продукт] купить
[тип товара] apple
```

**Tier 3 — Длинный хвост** *(низкий объём, высокая конверсия)*

Модель + условие покупки, кириллика без гео, самовывоз с уточнением точки. Минимум 5 запросов.

```
[продукт] в кредит [город]

[продукт] самовывоз севастополь

```
