# Service Info Block Builder (Block 3)

Генерирует готовый CS-Cart HTML-блок с плитками и отзывами для страниц ремонта Maxmobiles.

---

## Обязательный порядок работы (без исключений)

1. **Старт без генерации.** При первом обращении **не** создавать файл, **не** подставлять тексты из эталона или других страниц, **не** придумывать формулировки. Исключение — только техническое копирование CSS и секции отзывов из эталона **в момент финальной сборки**, после того как все пользовательские тексты получены.

2. **Сбор текстов по порядку.** Запросить у пользователя материалы **последовательно**, в одном логическом порядке с таблицей ниже:
   - сначала спросить: **какой заголовок H2 для этапов и какой текст у шагов 1, 2, 3, 4**;
   - после ответа — **какой заголовок H2 и какой текст для блока гарантии**;
   - затем — **для блока запчастей**;
   - затем — **для блока стоимости** (включая пункты со списком `-`, если нужны).
   Между шагами дожидаться ответа пользователя. Если в одном сообщении прислано сразу всё — допустимо, но **только если это явно полный комплект** по всем четырём блокам; при любой неполноте — уточнить **до** генерации.

3. **Генерация только после полноты.** HTML и файл `SLUG-block-3.html` собирать **только когда** есть все четыре блока пользовательских текстов (этапы, гарантия, запчасти, стоимость). Имя файла / SLUG: если пользователь не указал устройство для имени файла — **спросить** перед сохранением, не угадывать.

4. **Отзывы** по-прежнему не запрашивать — только копировать из эталона при финальной сборке.

---

## Обязательные входные данные

Используются **только** тексты, присланные пользователем в процессе шага выше. Таблица — что именно должно быть собрано:

| # | Что | Пример |
|---|-----|--------|
| 1 | **Заголовок H2 этапов** + **4 шага** (текст каждого) | `Как починить iPhone 14 Pro в 4 этапа?` |
| 2 | **Заголовок H2 гарантии** + **текст** | `Что по поводу гарантии?` |
| 3 | **Заголовок H2 запчастей** + **текст** | `Какие запчасти вы используете?` |
| 4 | **Заголовок H2 стоимости** + **текст** (может содержать список через `-`) | `Сколько стоит ремонт?` |

**Отзывы — всегда фиксированные** (не запрашивай, копируй из эталона).

---

## Процесс генерации

**Начинать этот раздел только после выполнения раздела «Обязательный порядок работы».**

Читай эталон: `Новые категории ремонта\iphone-15-pro-max-block-3.html`

### Шаги

1. **CSS** — копируй полностью без изменений.

2. **Секция этапов** — подставь H2 и тексты 4 шагов. Структура карточек не меняется.

3. **Плитка гарантии** — подставь H2 и текст. Иконка, цвета, бейдж «До 12 месяцев» — без изменений.  
   Если в тексте несколько абзацев — каждый в отдельный `<p>`.

4. **Плитка запчастей** — подставь H2 и текст. Иконка и бейдж «Все запчасти в наличии» — без изменений.  
   Если несколько абзацев — каждый в `<p>`.

5. **Плитка стоимости** — подставь H2 и текст.  
   Если в тексте есть список через `- `:
   ```html
   <ul class="mmb3-list">
     <li>Пункт 1</li>
     <li>Пункт 2</li>
   </ul>
   ```
   Остальной текст — в `<p>`.

6. **Секция отзывов** — копируй из эталона **без единого изменения** (все 4 карточки, ссылка, стили).

7. **Имя файла** — `SLUG-block-3.html`, где SLUG = транслит устройства через дефис.  
   Сохрани в `Новые категории ремонта\`.

---

## Что НЕ менять никогда

- Весь CSS блок (`.mmb3-*`)
- Структуру HTML (секции, классы, вложенность)
- Иконки SVG в плитках
- Бейджи в плитках гарантии и запчастей
- Секцию отзывов целиком (4 карточки + ссылка на Яндекс Карты)
- `margin-bottom: 40px` на `.mmb3`
- Тексты пользователя — **не редактировать**, вставлять как есть

## Формат входных данных от пользователя

Пользователь присылает текст в свободной форме. Пример:

```
Заголовок этапов: Как починить MacBook в 4 этапа?
1. Позвоните нам или оставьте заявку онлайн.
2. Привезите Mac в сервис.
3. Инженер проведёт диагностику и ремонт.
4. Заберите Mac с гарантийным талоном.

Гарантия — заголовок: Гарантия на ремонт MacBook
Гарантия — текст: Предоставляем гарантию до 12 месяцев...

Запчасти — заголовок: Какие запчасти мы используем?
Запчасти — текст: Используем только оригинальные компоненты...

Стоимость — заголовок: Сколько стоит ремонт MacBook?
Стоимость — текст: Цены фиксированные...
- Если нужной услуги нет в прайсе — цена обсуждается
- Возможен ремонт в кредит
```

Если пользователь прислал текст сплошным блоком — самостоятельно определи к какой секции относится каждый абзац по контексту.
