# Service Price Hydration (CS-Cart REST API)

Добавляет в `*-block-2.html` второй `<script>` блок. При каждой загрузке страницы делает
**один** API-запрос к CS-Cart, получает все товары категории с ценами и вариантами,
и обновляет `<span class="mmb2-price">` без изменения статической разметки таблицы.

---

## Принцип работы

**Static-first + JS hydration:**

- Статические цены в HTML остаются нетронутыми → SEO-индексация и fallback при ошибке
- JS делает **один** `fetch()` к CS-Cart REST API `/api/products/?cid=CAT_ID&get_options=true`
- Из ответа строится `priceMap`: `{ seo_name: { price, variants: { variantName: price } } }`
- Цены применяются к `<span class="mmb2-price">` по slug из `href` + имени варианта из текста строки
- **Кэш не используется** — цены загружаются свежо при каждой загрузке страницы

---

## API

```
GET https://maxmobiles.ru/api/products/?cid=CAT_ID&get_options=true&items_per_page=100
Authorization: Basic base64('servis-api@maxmobiles.ru:k57Z8R11085Y98m5l7s5JDJlaqsS37P3')
Accept: application/json
```

> **Критично**: без `Accept: application/json` CS-Cart может вернуть HTML вместо JSON — скрипт молча упадёт в `.catch()` и цены не обновятся.

Возвращает все товары категории. Каждый товар содержит:
- `seo_name` — slug (совпадает с хвостом `href`)
- `price` — цена (или цена дефолтного варианта)
- `has_options` + `product_options[].variants[].{ variant_name, modifier, modifier_type }`

`modifier_type: "A"` = абсолютная цена варианта (используем `modifier` как цену напрямую).

---

## Сопоставление строк таблицы с API

- `slug = href.replace(/\/$/, '').split('/').pop()` → ключ для поиска в `priceMap`
- Вариант: из текста `<a class="mmb2-svc-name">`:
  - `копия` → `'Копия'`
  - `оригинал` + `б/у` → `'Оригинал "б/у"'`
  - `оригинал` + `подлин` → `'Оригинал "подлинный"'`
  - `оригинал` → `'Оригинал'`
  - иначе → `''` (берётся `entry.price`)
- `isFrom` = есть ли `<span style="...ff9500...">` внутри `.mmb2-price`

## Процесс добавления в файл

### Шаг 1 — Найди CAT_ID для устройства

В CS-Cart категория устройства (например «iPhone 15 Pro Max ремонт»). ID можно взять:
- Из URL в панели CS-Cart при редактировании категории
- Или у пользователя (попроси CAT_ID)

Пример: iPhone 15 Pro Max → `CAT_ID = 2260`

### Шаг 2 — Прочитай эталон

Открой `Новые категории ремонта/iphone-15-pro-max-block-2.html` и найди блок
`<!-- JAVASCRIPT — гидратация цен через CS-Cart REST API -->` в конце файла.
Копируй его полностью.

### Шаг 3 — Вставь после существующего `</script>`

Блок вставляется **после** закрывающего `{/literal}</script>` первого скрипта
(функции `mmb2OpenModal` и `mmb2InitAccordion`). Всегда второй по порядку.

### Шаг 4 — Измени только CAT_ID

```javascript
var CAT_ID = 2260;  // ← замени на ID категории нового устройства
```

Всё остальное — не трогай. `CACHE_KEY` автоматически уникален через `window.location.pathname`.

### Шаг 5 — Ничего не меняй в разметке таблицы

Блок гидратации работает с **существующей** разметкой:
- читает `href` из `<a class="mmb2-svc-name">`
- определяет вариант из текста ссылки
- определяет признак «от» по наличию `<span style="...ff9500...">` внутри `.mmb2-price`

Никаких `data-*` атрибутов добавлять не нужно.

---

## Что НЕ менять при копировании в новый файл

- Функции `formatPrice`, `applyPrice`, `slugFromHref`, `applyAll` — не трогать
- Логику кэша (`readCache`, `writeCache`) — не трогать
- Логику детекции вариантов — не трогать
- Таймаут AbortController (8 000 мс) — не трогать
- `API_URL`, `API_AUTH` — не трогать
- Заголовки fetch: оба `'Authorization'` и `'Accept': 'application/json'` — не убирать
- sessionStorage/кэш — **не добавлять**. Цены должны грузиться свежо при каждом открытии страницы

Менять только: `CAT_ID`

---

## Совместимость с ручным пересбором цен

Оба подхода работают параллельно:
- **Ручной пересбор** (команда «пересобери цены») обновляет статические значения в HTML
- **JS гидратация** при каждой загрузке страницы перезаписывает их актуальными

Ручной пересбор полезен для поддержания SEO-актуальности.
JS гидратация — основной механизм актуальности цен для живых пользователей.
