# SEO Meta Analyzer — Maxmobiles

Анализирует текущие мета-теги, исследует SERP и конкурентов, строит семантический кластер и выдаёт финальные рекомендации H1 / title / description.

Гео-приоритет: **Севастополь** (primary) → **Крым** (secondary) → Россия (доставка, не основной).

---

## Workflow

### Шаг 1 — Входной аудит

Принять от пользователя:
- Текущие H1 / title / description (или «нет тегов»)
- Название категории или модели (например: «iPhone 17», «MacBook Pro M4», «AirPods Pro 3»)
- Тип страницы: категория / конкретная модель / б/у / аксессуар

Проверить текущие теги по чеклисту из `seo--meta-tags.mdc`:

| Параметр | Норма | Статус |
|---|---|---|
| H1 длина | 50–70 симв. | ✅/❌ |
| title длина | 50–65 симв. | ✅/❌ |
| description длина | 120–155 симв. | ✅/❌ |
| Гео в H1 | Севастополь ИЛИ Крым | ✅/❌ |
| Гео в title | Севастополь ИЛИ Крым | ✅/❌ |
| H1 ≠ title (не дубль) | разные формулировки | ✅/❌ |
| Гео не одинаковое в H1 и title | не оба «Севастополь» | ✅/❌ |
| Запреты | нет цен ₽, !, капслока | ✅/❌ |
| E-E-A-T сигналы | ≥ 2 USP в description | ✅/❌ |

Вывести таблицу аудита **до** начала исследования.

---

### Шаг 2 — SERP & семантика

Запустить **4 поиска параллельно**:

1. `"купить [Модель/Категория] Севастополь"` — основной коммерческий + гео
2. `"[Модель/Категория] цена Крым купить"` — ценовой + крымский гео
3. `"[Модель/Категория] купить интернет-магазин доставка Россия"` — доставочный кластер
4. `"[Модель/Категория] [текущий год] купить официально гарантия"` — коммерческий + доверие

**Цель**: найти реальные формулировки, которые используют пользователи и которые попадают в title/description конкурентов из топа органики.

---

### Шаг 3 — Конкурентный анализ

Из результатов поиска (Шаг 2) отобрать **3–5 органических конкурентов**:

- Брать только **интернет-магазины** с явными product/category страницами
- **Исключить**: Wildberries, Ozon, DNS, Citilink, М.Видео, Яндекс.Маркет, агрегаторы, Авито, отзывники, Wikipedia, новостные сайты

Для каждого конкурента (WebFetch) записать:

| Параметр | Что фиксировать |
|---|---|
| H1 | точный текст |
| title | точный текст |
| description | точный текст (если виден в сниппете) |
| Гео-модификаторы | как формулируют гео |
| USP в тегах | гарантия, доставка, Trade-In и т.д. |
| Паттерн title | шаблон структуры |

Вывести сводную таблицу конкурентов **перед рекомендациями**.

---

### Шаг 4 — LSI-кластер

Собрать семантику на основе Шагов 2–3 (внутренняя таблица, не выводить пользователю):

| Кластер | Запросы |
|---|---|
| Основной коммерческий | «купить [Модель] Севастополь», «[Модель] цена Крым» |
| Доставка | «доставка СДЭК», «доставка по России» |
| Условия покупки | «гарантия», «Trade-In», «рассрочка», «кредит» |
| Доверие | «с 2010 года», «официальный магазин», «15 лет» |
| LSI-хвосты | смежные слова из сниппетов конкурентов |

Определить **1–2 ключа** для H1, **1 ключ** для title, **2–3 USP** для description.

---

### Шаг 5 — GEO / AEO слой

**GEO (нейровыдача — Яндекс ИИ, ChatGPT Search, Gemini):**

- Description: первое предложение = прямой ответ на запрос (AI цитирует дословно)
- Структура: `[Модель] в Maxmobiles — [USP-1], [USP-2]. [Факт доверия]. [CTA].`
- Все 5 сущностей в description: бренд + гео + модель + действие + условие

**AEO (голосовой поиск, Featured Snippets):**

- Description должен отвечать на вопрос «Где купить [Модель] в Севастополе?» в первом предложении
- Формулировка естественная, не ключевой спам

---

### Шаг 6 — Рекомендации

Вывести финальный блок строго в формате из `seo--meta-tags.mdc`:

```
H1:          [текст]                              ([N] симв.)
title:       [текст]                              ([N] симв.)
description: [текст]                              ([N] симв.)
```

После блока — комментарий:

```
## Обоснование

**H1**: [какой ключ покрывает, почему выбрано это гео]
**title**: [паттерн, почему отличается от H1]
**description**: [какие USP включены, какие E-E-A-T сигналы, GEO-принцип]

## Изменения vs текущие теги
- H1: [что изменилось и почему]
- title: [что изменилось и почему]
- description: [что изменилось и почему]

## Семантическое покрытие
Запросы, которые теперь покрывают эти теги:
- [запрос 1]
- [запрос 2]
- ...
```

**Отклонения от стандарта** допустимы, если:
- Данные SERP показывают доминирующий паттерн, не совпадающий с шаблоном
- Конкурент в топ-1 использует другую структуру с очевидным результатом
- Ключевой запрос не вписывается в стандартную длину

При отклонении — явно указать: `⚠️ Отклонение от стандарта: [причина]`

---

### Шаг 7 — Альтернативные варианты (опционально)

Если есть неоднозначность (например, два равноценных ключа), вывести **2 варианта** с пометкой:

```
Вариант A (рекомендуемый):
H1: ...
title: ...
description: ...

Вариант B (если приоритет — доставка по России):
H1: ...
title: ...
description: ...
```

## Правила фильтрации конкурентов

Принимать в анализ:
- Региональные интернет-магазины техники
- Федеральные магазины с выраженной category page (re:Store, iPort и т.п.)
- Узкоспециализированные Apple-магазины

Отклонять:
- Маркетплейсы: Wildberries, Ozon, AliExpress, Яндекс.Маркет, СберМегаМаркет
- Агрегаторы цен: Price.ru, Яндекс.Поиск по товарам, market.yandex.ru
- Крупные сети-ретейлеры без своего домена: DNS, Citilink, М.Видео, Эльдорадо
- Контентные сайты: обзоры, новости, форумы, Wikipedia
