# Команда: Сервис баннер

Назначение: генерация баннер-блока (Block 1) для страниц ремонта Maxmobiles по стандарту проекта.

## Что использовать

- Rule: `service-banner-block.mdc`
- Skill: `service-banner-block-builder`
- Reference: `.cursor/skills/service-banner-block-builder/template-reference.md`
- Эталон (API-версия): `Новые категории ремонта/iphone-15-pro-max-block-1-api.html`

## Инструкция для агента

1. Прочитай и строго соблюдай:
   - `.cursor/rules/service-banner-block.mdc`
   - `.cursor/skills/service-banner-block-builder/SKILL.md`
   - `.cursor/skills/service-banner-block-builder/template-reference.md`
2. Бери за основу эталон `iphone-15-pro-max-block-1-api.html`, не блок без API.
3. Если у пользователя нет входных данных — запроси всё из таблицы скилла: заголовок, текст под заголовком, URL картинки, список услуг (название, URL товара, время, гарантия, опция при нескольких вариантах), **CAT_ID**. Уточни `page_id` и ID полей формы (`form_values[N]`), не выдумывай.
4. Выбери уникальный префикс CSS/JS по правилу скилла и замени все вхождения базового префикса эталона (включая ключи кэша API при необходимости согласно эталону и заменам из template-reference).
5. Массив `SERVICES`: `price` и `priceNum` только заглушки `'— ₽'` и `'—'`; в разметке `<span id="ПРЕФИКС-price">` тоже `—`. Комментарии в HTML/CSS/JS не добавлять.
6. Пропиши `CAT_ID`, собери `<select>` услуг, сохрани файл как `SLUG-block-1.html` в `Новые категории ремонта/` (SLUG по правилу скилла).
7. В чат выведи краткое подтверждение и путь к файлу.
