# Команда: Сервис таблица цен

Назначение: добавление или проверка JS-блока **гидратации цен** (CS-Cart REST API) в существующем файле таблицы цен `*-block-2.html`.

## Что использовать

- Skill: `service-price-hydration`
- Rule: `service-price-hydration.mdc`
- Эталон второго скрипта: конец файла `Новые категории ремонта/iphone-15-pro-max-block-2.html` (блок «JAVASCRIPT — гидратация цен через CS-Cart REST API»)

## Инструкция для агента

1. Прочитай и соблюдай:
   - `.cursor/skills/service-price-hydration/SKILL.md`
   - `.cursor/rules/service-price-hydration.mdc`
2. Целевой файл: указанный пользователем `Новые категории ремонта/<slug>-block-2.html` или попроси путь/устройство.
3. Уточни **CAT_ID** категории CS-Cart для этого ремонта (из админки или у пользователя). Не подставляй число без источника.
4. Если второго `<script>` с гидратацией ещё нет — скопируй блок целиком из эталона `iphone-15-pro-max-block-2.html` и вставь **после** первого `{/literal}</script>` (аккордеон/модалка). Это всегда второй скрипт по порядку.
5. Если блок уже есть — проверь соответствие эталону и заголовки `fetch`: **`Authorization`** и **`Accept: application/json`** обязательны.
6. В API-блоке меняй только **`CAT_ID`** под устройство; разметку таблицы и классы `.mmb2-price` / `.mmb2-svc-name` для гидратации не ломай.
7. Убедись, что для Smarty сохранены обёртки `{literal}...{/literal}` у обоих скриптов.
8. В чат выведи краткое подтверждение и путь к изменённому файлу.
