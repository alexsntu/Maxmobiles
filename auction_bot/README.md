# Auction Bot — Telegram-бот для аукционов

Бот для проведения **английских аукционов** (ставки растут) в Telegram-группах.

---

## Возможности

- Создание лотов через мастер-диалог (фото, описание, стартовая цена, шаг, длительность)
- Публикация лота в группу с фото и инлайн-кнопками
- Участники делают ставки двумя способами:
  - Нажатием кнопки «🔨 Сделать ставку» — бот пишет в личку
  - Ответом (Reply) на сообщение лота с суммой в числовом формате
- Автоматическое закрытие аукциона по таймеру
- **Защита от снайпинга**: ставка в последние 60 сек продлевает таймер на 60 сек
- Объявление победителя + уведомление в личку
- История ставок по кнопке «📊 История ставок»
- Управление лотами для администраторов (`/newlot`, `/lots`, `/cancel`)
- Восстановление активных аукционов после перезапуска бота

## Установка и настройка

### 1. Требования

- Python **3.11** или выше
- pip

### 2. Клонируйте / скачайте проект

Поместите папку `auction_bot/` на сервер или локальный ПК.

### 3. Установите зависимости

```bash
cd auction_bot
pip install -r requirements.txt
```

### 4. Создайте `.env`

Скопируйте `.env.example` и заполните своими данными:

```bash
cp .env.example .env
```

Откройте `.env` и укажите:

```env
BOT_TOKEN=1234567890:AAxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
ADMIN_IDS=123456789
GROUP_ID=-1001234567890
TIMEZONE=Europe/Moscow
```

**Как получить нужные данные:**

| Переменная | Как получить |
|---|---|
| `BOT_TOKEN` | Создайте бота у [@BotFather](https://t.me/BotFather), команда `/newbot` |
| `ADMIN_IDS` | Узнайте свой Telegram ID у [@userinfobot](https://t.me/userinfobot). Несколько ID — через запятую |
| `GROUP_ID` | Добавьте бота в группу, перешлите любое сообщение из группы [@userinfobot](https://t.me/userinfobot) — он покажет ID группы |

### 5. Настройте бота в группе

1. Добавьте бота в вашу Telegram-группу
2. Выдайте боту права **администратора** в группе (нужны права: отправка сообщений, редактирование сообщений)
3. Напишите боту `/start` в **личку** — это обязательно, иначе бот не сможет отправить вам уведомления

### 6. Запуск

```bash
python bot.py
```

Для фонового запуска на сервере (Linux):

```bash
nohup python bot.py > bot.log 2>&1 &
```

Или через `screen`:

```bash
screen -S auction_bot
python bot.py
# Ctrl+A, затем D — выход из screen без остановки бота
```

## Команды

### Для всех пользователей
| Команда | Описание |
|---|---|
| `/start` | Приветствие и инструкция |
| `/help` | Справка по правилам аукциона |

### Только для администраторов (в личке бота)
| Команда | Описание |
|---|---|
| `/newlot` | Запустить мастер создания нового лота |
| `/lots` | Список активных лотов с управлением |
| `/cancel <id>` | Отменить лот по ID (например `/cancel 5`) |

## Структура файлов

```
auction_bot/
├── bot.py              # Точка входа
├── config.py           # Конфигурация из .env
├── database.py         # SQLite: схема и все запросы
├── scheduler.py        # APScheduler: авто-закрытие лотов

├── states.py           # FSM-состояния
├── utils.py            # Форматирование сообщений
├── handlers/
│   ├── common.py       # /start, /help
│   ├── admin.py        # Управление лотами
│   └── bidding.py      # Обработка ставок
├── .env                # Ваши секреты (не коммитить!)
├── .env.example        # Шаблон конфигурации
├── requirements.txt    # Зависимости Python
└── auction.db          # SQLite БД (создаётся автоматически)
```

## Частые вопросы

**Бот не может написать победителю в личку** — пользователь должен сам написать боту `/start` в личке хотя бы один раз.

**Как узнать ID группы?** — Добавьте бота, затем перешлите любое сообщение из группы боту [@userinfobot](https://t.me/userinfobot). ID группы будет отрицательным числом вида `-1001234567890`.

**Несколько администраторов** — укажите их ID через запятую: `ADMIN_IDS=111,222,333`.

**Бот перезапустился во время аукциона** — при старте бот автоматически восстанавливает все активные лоты и досрочно закрывает те, время которых уже истекло.
